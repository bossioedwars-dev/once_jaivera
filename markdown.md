# Taller componentes del internet o la red

## 1. ¿Cómo funciona internet?

### La finalidad de los protocolos **TCP/IP**

* **IP:** Concede una identidad numérica decimal con puntos exclusiva a cada equipo vinculado a la red.
* **TCP:** Funciona para distribuir la información en fragmentos reducidos y clasificarlos según el grupo al que correspondan (documentos, correos, imágenes, etc.).

---

### La utilidad del sistema **DNS**

Actúa como el **intérprete** para que cualquier usuario logre alcanzar la IP sin la necesidad de recordar puntos y cifras. Basta con añadir términos sencillos de reconocer para hallar la ubicación en la red. 

> **Ejemplo:** Escribir "Google", acceder al portal web y entrar a su dirección **IP** sin tener que grabarla en la memoria.

---

### El papel del protocolo **DHCP**

El **DHCP** otorga de forma automática una dirección IP libre a tu equipo para que no debas establecerla de manera manual. Esto evita ajustes técnicos complicados y permite que se acceda directamente al sitio o se realice la tarea deseada al instante.

---

## 2. Protocolo HTTP y la Web 

### Las partes fundamentales de una URL

1. **Esquema (Protocolo):** `https` (representa la modalidad de transmisión protegida).
2. **Dominio:** `www.ejemplo.com` (la denominación del sitio).
3. **Ruta:** `/productos` (la "subcarpeta" o área particular dentro de la web).

---

### La distinción esencial entre los verbos HTTP

* **GET:** Se utiliza para **solicitar** datos. Al ingresar a un blog para consultar contenido, estás ejecutando un GET. Generalmente, los datos se visualizan en la URL.
* **POST:** Se emplea para **transmitir** datos. Al introducir tu cuenta y clave o cargar un archivo, utilizas POST. Resulta más reservado porque la información viaja "oculta" en el cuerpo del envío.

---

### El sentido y objetivo de los códigos de respuesta habituales

Código Significado Descripción 

**200 OK**  Éxito total  Tu sitio está listo y cargó correctamente. 

**301 Moved**  Traslado La web se movió a una ubicación distinta (redirección). 

**404 Not Found**  No encontrado  El servidor opera bien, pero lo que buscas no está. 

**500 Error**  Falla interna  El servidor colapsó; es un error del sistema, no tuyo.
